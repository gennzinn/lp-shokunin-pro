# HTML技術仕様

05_template_html.md のテンプレートを土台に、この仕様を満たすHTMLを生成する。
**この仕様はデザインの希望より優先する(絶対条件)。**

## 絶対条件(全LP共通)

1. **単一ファイル完結**:HTML/CSS/JSをすべて1つのindex.htmlに書く。
   外部参照はGoogle Fontsと、ユーザーが明示的に指定した計測タグのみ。
2. **画像ファイルを作らない・創作しない**:装飾はインラインSVGとCSSグラデーションで表現。
   ユーザーが実在URLを渡した場合のみ`<img>`を使い、必ず
   `onerror="this.style.display='none'"` を付けて消えても崩れないようにする。
3. **幅375pxで横スクロール禁止**:複数カラムのgridは
   `@media(max-width:860px)`前後で1カラムに畳む。固定幅指定を避け、max-widthと%で組む。
4. **JS無効でも全文表示**:`<head>`内で
   `document.documentElement.classList.add('js')` を実行し、
   フェードイン用の非表示スタイルは必ず `.js .reveal{...}` 配下に書く。
   IntersectionObserverで表示時に`.on`付与、未対応ブラウザは即`.on`。
   `@media(prefers-reduced-motion:reduce)`でアニメーション無効化。
5. **CTAはゴール1つに統一**:ヘッダー・ヒーロー・最終セクションの最低3箇所。
   電話がゴールの場合は `href="tel:0000000000"`(ハイフンなし)。
6. **色は`:root`のCSS変数で一元管理**(テーマの色をそのまま変数化)。
7. **日本語コメント**をセクション区切りに入れる(`<!-- ========== 料金 ========== -->`)。
8. **文字コードはUTF-8**、`lang="ja"`。

## メタ情報

- `<title>`:「サービス名 | 会社名」形式。32字以内目安
- `meta description`:80〜120字。地域名+サービス+ベネフィット
- OGP:og:site_name / og:title / og:description / og:url / og:type を必ず入れる。
  **og:imageは実在が確認できるURLを渡された場合のみ**設定。なければタグごと省略
- favicon:SVGのdata URIで自作(サービスの頭文字1〜2字をテーマ色の角丸背景に載せる)
- `meta name="format-detection" content="telephone=no"`(電話CTA以外の番号の誤リンク防止)

## 計測タグの扱い

- ユーザーが指定した場合のみ、指定されたIDで`<head>`の先頭に入れる
- **他のクライアントのタグID・フォームURLを流用することは絶対禁止**
- 指定がなければ何も入れず、「公開後に計測タグを入れる場合はhead内に追記できます」と納品時に一言添える

## アクセシビリティ(最低ライン)

- 文字色と背景のコントラストを確保(薄グレー文字を14px未満で使わない)
- `<img>`には内容が伝わるalt。装飾SVGは `aria-hidden="true"` または role/aria-label
- リンクテキストは「こちら」でなく行き先がわかる文言
- 見出しはh1→h2→h3の順序を飛ばさない(h1はヒーローのキャッチコピー1つだけ)

## テンプレートの使い方

- 05のテンプレートの【 】プレースホルダを案件の内容で置き換える
- セクションは案件に合わせて**増減してよい**(悩み提起が不要な指名買い商材、FAQ不要など)。
  ただし「ヒーロー→(課題)→提供価値→信頼→料金→行動」の説得の流れは崩さない
- テーマ(03)の色・フォントを`:root`変数とGoogle Fonts読み込みに反映する
- クラス名・構造はテンプレートを維持する(品質の再現性のため、思いつきの別実装をしない)

## 出力形式

- 完成したHTMLを**1つのコードブロック**で出力(前後の説明は最小限)
- 生成直後に 06_quality_check.md のセルフチェックを実行して報告する
