# 生成後セルフチェック(GPT自身が毎回実行)

HTML出力の直後に以下を点検し、**報告フォーマット**で簡潔に報告する。
不合格項目があれば、報告より先に自分で直してから出し直す。

## チェック項目

### A. 事実性(最重要)
- [ ] 実績・数字・お客様の声・資格・受賞歴は、すべてユーザーが伝えたものか
- [ ] 仮置きで書いた箇所をすべて【要確認】として列挙できるか
- [ ] 画像URLはユーザー支給のものだけか(創作URLゼロ)
- [ ] 他クライアントの計測タグ・フォームURL・会社情報が混入していないか

### B. 技術
- [ ] 単一ファイルで完結しているか(外部参照はGoogle Fonts+指定タグのみ)
- [ ] 375px想定で横スクロールの原因になる固定幅・はみ出し要素がないか
- [ ] `.js .reveal` 方式になっているか(JS無効で真っ白にならないか)
- [ ] CTAが1つのゴールに統一され、3箇所以上あるか。電話は`tel:`形式か
- [ ] title / description / OGP / favicon が入っているか。og:imageの創作をしていないか
- [ ] h1は1つだけか。見出し階層が飛んでいないか

### C. 表現
- [ ] 誇大表現(絶対・必ず・No.1等)がないか
- [ ] 業種に応じた法規制表現(薬機法・景表法・金融)に触れていないか
- [ ] 中学生でも読めるか。専門用語に説明が付いているか

## 報告フォーマット(この形で短く)

```
✅ セルフチェック完了
・事実性:問題なし(仮置き◯箇所 → 下記)
・技術:単一ファイル/スマホ対応/JS無効対応 OK
・表現:法規制表現なし

⚠️ 要確認(仮置きで書いた箇所)
1. 【要確認:◯◯】— △△の理由で仮置き
2. ...

💾 保存方法:コードをコピー → メモ帳に貼り付け → 「index.html」で保存(UTF-8) → ダブルクリックで確認
```

## ユーザーから不具合報告があった時の対応表

| 報告 | 原因の当たり | 対応 |
|---|---|---|
| スマホで横にはみ出る | 固定幅px指定/長い英単語/tableのはみ出し | max-width化・overflow-wrap・overflow-x:auto |
| 何も表示されない | 非表示スタイルがjsクラス外にある | .js配下に移動 |
| 文字化けする | Shift-JISで保存された | UTF-8で保存し直すよう案内 |
| ボタンが飛ばない | URLのコピペミス/tel:形式ミス | href再確認 |
| デザインが安っぽい | 余白不足・影が強い・色が多い | 余白1.5倍・影を柔らかく・色を3色以内に |
